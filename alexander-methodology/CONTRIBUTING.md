## Code of Conduct

### Core Principles

1. **Respect**: Treat all researchers with dignity and respect
2. **Honesty**: Never fabricate or falsify data or results
3. **Collaboration**: Share knowledge and help others succeed
4. **Openness**: Embrace diverse perspectives and unconventional ideas
5. **Ethics**: Do no harm in pursuit of knowledge

### Unacceptable Behavior

- Harassment or discrimination of any kind
- Plagiarism or intellectual property theft
- Deliberate misinformation or sabotage
- Personal attacks or trolling
- Violation of privacy or consent

### Enforcement

Violations will result in:
1. Warning and guidance for minor issues
2. Temporary suspension for repeated issues
3. Permanent ban for serious violations

Report violations via GitHub issues or private message to council.

## Contribution Guidelines

### Code Style

**Python**:
- Follow PEP 8
- Use type hints
- Add docstrings to all functions
- Maximum line length: 100 characters

**Bash Scripts**:
- Use `#!/bin/bash` shebang
- Include script description at top
- Check for errors with `set -e`
- Quote variables properly

**YAML**:
- 2-space indentation
- Use consistent formatting
- Add comments for complex sections

**Markdown**:
- Use headings hierarchically
- Add table of contents for long documents
- Include code examples where relevant
- Keep paragraphs concise

### Commit Messages

Format: `Type: Brief description`

**Types**:
- `Add:` New feature or content
- `Fix:` Bug fix
- `Update:` Modify existing feature
- `Remove:` Delete code or files
- `Docs:` Documentation only
- `Refactor:` Code restructuring
- `Test:` Add or modify tests

**Example**:
```
Add: Voynich manuscript frequency analysis script

Implements statistical frequency analysis for the Voynich manuscript
text. Includes visualization of letter distributions and pattern
detection algorithms.

Closes #123
```

### Pull Request Template

```markdown
## Description
Brief description of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation
- [ ] Research contribution
- [ ] Infrastructure improvement

## Related Issues
Fixes #(issue number)

## Testing
How did you test your changes?

## Checklist
- [ ] Code follows style guidelines
- [ ] Self-review completed
- [ ] Commented complex code
- [ ] Documentation updated
- [ ] No breaking changes (or documented)
- [ ] Tested locally
```

### Review Process

1. **Automated Checks**: CI/CD runs tests and linting
2. **Peer Review**: Community members provide feedback
3. **Maintainer Review**: Core team final approval
4. **Merge**: Changes integrated into main branch

Expect feedback! It's a collaborative process.
