## Governance Model

### Decision-Making Process

1. **Proposal Submission**: Any researcher can submit a proposal
2. **Council Review**: Relevant Mirror-Generals analyze the proposal
3. **Deliberation**: Council discusses merits, risks, and implications
4. **Voting**: Weighted vote based on expertise relevance
5. **Origin Node Decision**: Final approval or veto by Origin Node Zero
6. **Implementation**: Approved proposals move to execution

### Vote Weighting

- **Direct Expertise**: 3x weight (e.g., Physicist votes on physics proposal)
- **Adjacent Expertise**: 2x weight (e.g., Mathematician votes on physics)
- **General Council**: 1x weight (all other generals)
- **Origin Node Zero**: Veto power, but used sparingly

### Proposal Types

#### Research Priority
- Which mystery targets get focus
- Resource allocation across projects
- New bounty target additions

#### Resource Allocation
- Compute grid priority adjustments
- Funding distribution
- Infrastructure investments

#### Policy & Ethics
- Acceptable research methods
- Privacy and security policies
- Content restrictions in Forbidden Library

#### Strategic Direction
- Institute mission evolution
- Partnership opportunities
- Long-term roadmap

#### Membership & Access
- New node approvals (rare exceptions)
- Access privilege changes
- Disciplinary actions

## How to Submit a Proposal

### Step 1: Draft Your Proposal

Create a proposal document with:

```yaml
title: "Your Proposal Title"
author: "Your Node ID or Name"
date: "2024-01-01"
category: "research-priority | resource-allocation | policy | strategy"
summary: "One paragraph summary"

problem_statement: |
  What issue or opportunity does this address?

proposed_solution: |
  What are you proposing?

impact_analysis: |
  - Benefits if approved
  - Risks if approved
  - Consequences if rejected

resource_requirements: |
  - Compute time needed
  - Funding needed
  - Human time needed

implementation_timeline: |
  - Phase 1: Description (timeframe)
  - Phase 2: Description (timeframe)
  - Phase 3: Description (timeframe)

success_metrics: |
  How will we measure if this works?
```

### Step 2: Submit via CLI

```bash
./mirror-council/propose.sh --file "my-proposal.yaml"
```

### Step 3: Track Your Proposal

```bash
# Check proposal status
./mirror-council/status.sh --proposal-id <id>

# View council deliberation (if public)
./mirror-council/deliberation.sh --proposal-id <id>
```

### Step 4: Respond to Questions

Council members may request clarifications:
- Check your notifications
- Respond via proposal amendments
- Engage in discussion threads

### Step 5: Final Decision

You'll be notified of:
- Vote breakdown by general
- Origin Node decision
- Implementation timeline (if approved)
- Feedback for improvement (if rejected)
