### Documentation Standards

When writing documentation:
- **Clear and concise**: Get to the point quickly
- **Examples included**: Show, don't just tell
- **Multiple perspectives**: Technical + conceptual + practical
- **Accessible language**: Avoid unnecessary jargon
- **Well-structured**: Use headings, lists, and formatting
- **Visual aids**: Include diagrams where helpful
- **Links**: Connect related documentation
