## Operational Protocols

### When Collapse Is Detected
1. Log the event with full context (concept, timestamp, quadrants engaged)
2. Capture the externalization (agent, dashboard, or physical skill)
3. Schedule stress-test within 24 hours
4. Update productivity metrics

### When Collapse Fails
1. Identify which quadrant is missing or weak
2. Suggest concrete actions to strengthen that quadrant
3. Flag for Dom's attention in daily reflection
4. Track how long the concept remains in superposition

### When Performance Degrades
1. Check for physiological factors (sleep, stress, load)
2. Review recent collapse events for incomplete integrations
3. Assess if too many parallel streams are active
4. Recommend consolidation or externalization
